# 管理员通知功能说明

## 功能概述
点菜小程序现在已经实现了管理员通知功能，当用户下单后会自动将订单信息推送给管理员。

## 功能特性

### 1. 自动通知
- 用户在购物车结算下单后，系统会自动发送订单通知给管理员
- 通知包含订单的详细信息：订单号、菜品清单、价格、状态等

### 2. 多种通知方式
- **云函数通知**：通过微信云开发云函数发送通知
- **模板消息通知**：通过微信模板消息发送通知
- **本地存储通知**：将通知保存到本地存储（模拟功能）

### 3. 管理员页面
- 专门的管理员通知查看页面
- 显示未读通知数量标记
- 支持查看订单详情
- 支持批量操作（清空通知、全部标记已读）

## 配置说明

### 管理员配置文件
配置文件位置：`config/admin.js`

```javascript
const ADMIN_CONFIG = {
  // 管理员用户ID列表（微信用户的openid）
  adminOpenIds: [
    // 请在此处添加管理员的openid
    // 示例: 'oX1234567890abcdef'
  ],

  // 管理员手机号列表（可选，用于短信通知）
  adminPhones: [
    // 请在此处添加管理员手机号
    // 示例: '13800138000'
  ],

  // 通知方式配置
  notification: {
    // 是否启用模板消息通知
    enableTemplateMessage: false,
    
    // 模板消息ID（需要在微信公众平台申请）
    templateId: '',
    
    // 是否启用云函数通知
    enableCloudFunction: true,
    
    // 云函数名称
    cloudFunctionName: 'sendOrderNotification'
  }
}
```

### 配置步骤

1. **获取管理员openid**
   - 管理员需要登录小程序获取自己的openid
   - 将openid添加到 `config/admin.js` 的 `adminOpenIds` 数组中

2. **配置云函数通知**
   - 在微信云开发中创建云函数 `sendOrderNotification`
   - 云函数需要实现将订单通知发送给管理员的逻辑
   - 设置 `enableCloudFunction: true`

3. **配置模板消息通知（可选）**
   - 在微信公众平台申请模板消息
   - 获取模板ID并填入 `templateId`
   - 设置 `enableTemplateMessage: true`

## 使用说明

### 访问管理员页面
1. 打开小程序首页
2. 点击"管理员"功能入口
3. 进入管理员通知页面

### 查看通知
1. 在通知列表中可以看到所有订单通知
2. 未读通知会显示红色边框和红点标记
3. 点击通知可以查看订单详情
4. 查看后通知会自动标记为已读

### 管理通知
- **清空所有通知**：点击底部"清空所有通知"按钮
- **全部标记已读**：点击底部"全部标记为已读"按钮

## 核心功能实现

### 订单通知发送
在 `pages/cart/cart.js` 的结算方法中：
```javascript
// 发送订单通知给管理员
notification.sendOrderNotification(order)
```

### 通知工具类
`utils/notification.js` 提供了以下方法：
- `sendOrderNotification(order)` - 发送订单通知
- `getUnreadNotificationCount()` - 获取未读通知数量
- `markNotificationAsRead(id)` - 标记通知为已读

## 注意事项

1. **云开发配置**
   - 需要在小程序管理后台启用云开发
   - 需要创建相应的云函数

2. **模板消息申请**
   - 模板消息需要在微信公众平台申请
   - 用户需要授权才能接收模板消息

3. **管理员权限**
   - 目前所有用户都可以访问管理员页面
   - 建议添加管理员身份验证逻辑

4. **数据持久化**
   - 当前通知数据保存在本地存储
   - 生产环境建议使用数据库存储

## 扩展建议

4. **数据统计**
